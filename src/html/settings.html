• Item 3-2
  • 何度も繰り返し使用するtailwindクラスの組み合わせは@applyディレクティブで部品化する。
  • /src/css/tailwind.cssに記述
.axpt_btn { @apply bg-yellow-300 text-black font-bold py-2 px-4 rounded; }
ただし、この方法は、 スタイルの再利用:https://tailwindcss.com/docs/reusing-styles で語られている通り、
  • 都度、重複しないクラス名を考える必要が生まれる
  • HTML単体で完結していたコードが、複数のファイルを参照しないと理解できなくなる
  • 改造で変更を加える場合に、使用箇所すべての影響調査が必要になる
  • tailwindの仕様上、@applyでコードをまとめたほうがファイルサイズが大きくなる
というように、tailwind導入のメリットや意義を失うため、非推奨とされている。 よって、どうしても必要と判断される場合のみの最終手段とする。 なお、大規模プロジェクトではこの問題をスマートに解決するためのアンサーとしてReactやVueが用いられている。
